# Vision

## Problem Statement

Developers spend too much time typing. Voice coding can accelerate workflows, improve accessibility, and enable "vibe coding" - describing intent naturally rather than writing syntax.

## Vision Statement

**Voice OpenCode** aims to be the most accessible, portable, and modular voice interface for OpenCode AI coding agents.

## Core Principles

1. **Portability First** - Works on any platform without complex setup
2. **Modularity** - Swap STT/TTS engines without rewriting code
3. **Simplicity** - One command to start voice coding
4. **Accessibility** - Helps developers with RSI or typing limitations
5. **Community-Driven** - Built by developers, for developers

## Target Users

- Developers who want hands-free coding
- Developers with accessibility needs (RSI, repetitive strain)
- Power users who dictate faster than they type
- Anyone exploring "vibe coding" workflows

## Roadmap

### Phase 1: Foundation
- Cross-platform CLI (Bun)
- Basic STT/TTS adapters
- OpenCode API integration
- Local processing option

### Phase 2: Web UI
- Browser-based interface
- Real-time streaming
- Mobile-friendly

### Phase 3: Mobile
- Android/iOS companion
- Push notifications
- Remote session control

## Success Metrics

1. Number of active users
2. Time to first voice command
3. Number of STT/TTS engine adapters
4. Community contributions
5. Accessibility compliance

## Values

- **Open Source**: Free for everyone
- **Privacy**: Local processing option
- **Choice**: Multiple engine options
- **Quality**: Reliable, responsive voice interaction

## Values vs. Requirements

| Value | Requirement |
|-------|-------------|
| Privacy | Local STT/TTS option |
| Accessibility | Keyboard-free operation |
| Choice | Modular engine architecture |
| Portability | Cross-platform CLI |
